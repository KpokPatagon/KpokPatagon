Provides queue services using RabbitMQ services.

Example

Service configuration:

Queue services are configured by default within the "KpokPatagon:Queue" section in appsettings.json, but the actual key can be specified, if required, as an argument in the registration method.


             "KpokPatagon": {
               "Queue": {
                 "Url": "amqp://localhost:5672"
               }
             }

The configuration for both supported services is the same, the service Url that must include: the protocol, the host name of IP address and if required, port, user and password as required by each service.

Service registration:

The queue service must be registered with DI within the "configure service" method:


             services.AddKpokPatagon(<configuration>)
                     .AddRabbitQueueService();

Service usage:

Once registered just inject the service where required:


             readonly IQueusService _queueService;
             
             public MyClass(IQueueService queueService) 
             { 
                 _queueService = queueService;
             }
             
             async Task PublishMessageAsync(MyMessage message)
             {
                 await _queueService.PublishAsync("my-queue", message);
             }
             
             async Task ConsumeAsync()
             {
                 await _queueService.ConsumeAsync<MyMessage>("my-queue", 
                     (msg) => { /* Message procesing... */ },
                     (exception) => { /* Handle errors */ }
                 );
             }

ConsumeAsync

Setup a callback to be invoked when a new message is available on the queue.

If the message is successfully deserialized it is Ack before executing the onMessage callback; If an error occurred the message is Nack, without requeue, and the onError callback, if available, is executed

Signature:
public Task ConsumeAsync<T>(queue, onMessage, onError, serializer, cancellationToken)
Parameters
Name Type Summary
queue String The name f the queue.
onMessage Action<T> The callback action to process the message.
onError Action<Exception> An error callback action called when an error occurred while processing a message.
serializer IMessageSerializer A service to serialize the message.
cancellationToken CancellationToken A cancellation token.
